常见问题

为什么不用API就会失败？
Google 和 Bing 检测到来自服务器/程序的直接请求时，会弹出验证码（CAPTCHA）来区分人和机器。 因为我们的工具是程序（不是浏览器），无法完成"选出所有红绿灯"这类人机验证，所以被拦截。

使用官方 API 则不同——API 是 Google/Bing 官方提供的程序化接口， 他们允许通过 API 访问，只是有每天的调用次数限制。这就是为什么 API 从不被拦截。
DuckDuckGo 为什么不会被拦截？
DuckDuckGo（DDG）本身就是隐私搜索引擎，对爬虫宽容。我们使用的 ddgs 库底层用 模拟浏览器指纹的 TLS 库（模仿 Chrome/Edge 的加密握手特征）， 让 DDG 以为是真实浏览器在访问，所以不会触发验证码。

此外，DDG 调用的是 Bing 的搜索结果（DDG 自己不爬网页），所以结果质量接近 Bing。 这是目前最稳定且完全免费的方案。
免费额度用完了怎么办？
Google：100次/天。如果每天用量很大，可以在 Google Cloud Console 申请提额或付费。
Bing：1000次/月。Bing 的 F0 层不支持提额，超出后需要升级到 S1 付费层（约$7/1000次）。

其实日常使用 100次/天 绰绰有余——你搜索一次只需要调用1次API（搜索本身）， 后续的网站访问和分析不消耗API额度。DuckDuckGo 更是不限次数。
多个搜索引擎可以同时用吗？
可以！勾选多个引擎后，工具会依次调用每个引擎搜索，然后合并去重。 建议日常只用 DuckDuckGo，如果需要对特定国家做更精准的搜索，再开启 Bing API 或 Google API。
工具填的密钥安全吗？会被上传吗？
密钥保存在你的电脑本地（~/.number_finder_settings.json）， 只会发送到 Google/Bing 的官方 API 服务器，不会被上传到任何第三方。 代码完全开源，你可以自行审查 number_finder.py 中的 search_google_api 和 search_bing_api 函数。
